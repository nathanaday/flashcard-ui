

## List of True Facts

- TCP SYN segement does not carry any data
- The initial TCP sequence number is chosen at random (prevents spoofing)
- TCP allows the last segment in the TCP handshake to carry data
- A TCP sender uses slow start to probe the condition of the network because it does not know if it is congested 

## List of Example T/F

- A TCP sender uses slow start because it thinks the network is congested (FALSE)
